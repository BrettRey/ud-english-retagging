## 2026-04-14

- **Parallel layer, not overwrite.** Keep original UPOS/XPOS/DEPREL; add BR_CAT/BR_SUBTYPE/BR_STATUS in MISC or sidecar. Reason: safety, reversibility, and ability to compare UD vs CGEL tags.
- **Phase 1 scope: closed-class items only.** Pronouns/determinatives, temporal nouns, fused heads, interrogatives. Not attempting full reannotation. Reason: tractable, rule-based, paper-grounded.
- **Separate project from inventory/theory repo.** Different data format, outputs, scale, and revision cycle. Source categories come from the inventory work; this project operationalises them.
- **MVP narrowed to pronoun/determinative retagging on EWT dev.** The temporal-noun assumption fails on EWT, and location/direction items are a distinct harder problem. Reason: keep the first executable pass aligned with the evidence from the pilot corpus.
- **Sidecar TSV is the primary working format.** The MVP will not patch CoNLL-U or add MISC annotations directly. Reason: EWT already uses MISC heavily, and sidecars are easier to diff, audit, and regenerate.
- **Zero-extra-dependency Python for the first pass.** Do not assume `conllu` is installed; do not require YAML for the rules. Reason: the current environment does not have `conllu`, and CSV is sufficient for an inspectable rule table.
- **Operational pipeline reduced to rule table + retagger + audit.** Candidate extraction, patch writing, and a separate review queue are unnecessary as standalone components in the MVP. Reason: the pilot scope is small enough for a simpler architecture.
- **Relative `that` is auto-tagged as `subordinator`.** It is not a pronoun or determinative in the target analysis. Reason: this is a clear CGEL/UD disagreement and there is no value in carrying it as a review-only known error.
- **UD `SCONJ that` and `whether` are auto-tagged as `subordinator`.** Declarative/content `that` and interrogative `whether` belong to CGEL's subordinator class rather than the general preposition-with-clausal-complement class. Reason: this is a clear categorial decision and should not be buried inside the blanket `SCONJ -> preposition` rule.
- **Clear complement-clause `if` is auto-tagged as `subordinator`.** The current heuristic promotes `if` when the head of the `if`-clause is a `ccomp`, and also in coordinated complement-clause uses where the clause head is `conj` under a `ccomp`, leaving sentence-initial conditionals and other residual cases in the general preposition bucket for now. Reason: complement `if` is part of the CGEL subordinator class, but broader `if` disambiguation still exceeds safe token-local heuristics.
- **[Superseded] `so/SCONJ` stays review-visible.** It is no longer silently auto-tagged as `preposition[clausal]`. Reason at the time: this item seemed theoretically unsettled in the current scheme, and hiding it in the blanket `SCONJ -> preposition` rule created false certainty.
- **[Superseded] `along with` is surfaced as a review-visible complex-marker pattern.** The current rule flags `along` when it is immediately followed by `with`, without committing to a final analysis. Reason at the time: the user identified complex markers as unresolved, and this pattern was a real blind spot in the earlier sidecar outputs.
- **Clear coordinators are auto-tagged as `coordinator`.** The current auto set is `and`, `but`, `or`, `nor`, `plus`, and slash when UD uses `/` as a `cc` marker. Reason: these are clear members of the coordinator class in the target analysis and are easy to identify safely with token-local rules.
- **[Partially superseded] Marginal coordinator candidates stay review-visible unless the evidence is cleaner.** `yet`, `as well as`, `rather than`, and `et` were initially surfaced as explicit review items where UD gave a coordinator-like cue (`cc`, `ExtPos=CCONJ`, etc.), rather than being auto-normalized. Reason at the time: they seemed category-borderline or construction-specific, and the current rule table was token-local rather than sequence-aware.
- **`both`, `either`, and `neither` remain determinatives.** They are not to be reclassified as coordinators. Any marker-like use in correlative coordination is functional rather than categorial for this project. Reason: in the target analysis they are pure determinatives, and the current determiner rules already handle them correctly.
- **Marker-like `both`, `either`, and `neither` are auto-tagged as determinatives even outside UD `DET/PRON`.** The current rule table now covers their `CCONJ` and `ADV` uses directly. Reason: these are categorially determinatives in the target analysis, and leaving the non-`DET`/`PRON` uses outside the sidecar created a real coverage hole.
- **UD `SCONJ` is retagged as `preposition` with clausal complements.** Reason: this is one of the next major CGEL/UD mismatch classes and can be handled with simple explicit rules.
- **Infinitival `to` is auto-tagged as `verb[auxiliary_infinitival]`.** Reason: in the target analysis it is a highly defective auxiliary verb, and treating it explicitly keeps it out of the generic `PART` bucket.
- **Infinitival-clause `for` is auto-tagged as `subordinator`.** Restrict the rule to `for` marking `VerbForm=Inf` clause heads. Reason: this captures the CGEL analysis for `for NP to VP` without collapsing gerundial `for` uses like `Thanks for thinking` into the same category.
- **UD `n't` is not a lexical retagging target.** In the target analysis it is a negative morpheme on an auxiliary verb, not a separate word. Reason: the current pipeline is a token-level lexical retagger, so `n't` belongs to a later morphology/tokenization layer rather than this word-category layer.
- **The `ADV -> preposition` layer now includes the broader EWT set.** Items like `back`, `about`, `on`, `up`, and `down` are treated as prepositions when UD tags them `ADV`; noun and verb uses remain untouched because the rule is UPOS-specific. Reason: these items are not adverbs in the target analysis, and broadening the enumerated set increases coverage without adding review noise.
- **Superseding decision on the `ADV -> preposition` layer: expand it with appendix-backed items that are actually attested in the covered corpora.** `out`, `in`, `over`, `ago`, `before`, `once`, `forward`, `ahead`, `apart`, `hence`, and `forth` are now auto-tagged as `preposition[intransitive]` when UD tags them `ADV`. Reason: the `Language Landscapes` appendix confirms the category inventory, and a corpus scan showed these are the highest-value remaining `ADV` mismatches in the currently covered overt-form corpora.
- **Superseding decision on `n't`: represent it in the sidecar as `morpheme[negative_enclitic]`.** Reason: although the target analysis is morphological rather than lexical, the current pipeline still needs an explicit category for the separate UD token, and encoding it this way keeps the sidecar aligned with the analysis without pretending it is an adverb or particle in the output layer.
- **Numeratives now follow the current paper-level analysis.** Word-form cardinals are determinatives, magnitude-factor cardinals are determinatives, fused-head numeral uses are determinatives, ordinals are adjectives, and fractionals are nouns. Reason: the rule table should reflect the paper's category decisions before extending beyond EWT.
- **Negative and reciprocal `one` are auto-tagged as pronouns.** `no one` and reciprocal `one another` are ordinary pronoun cases, so they should not remain in fallback review. Reason: these are clear CGEL analyses and removing them materially reduces residual review burden in EWT.
- **The active rule table is `rules/cgel_retagging.csv`.** The project now covers coordinators, subordinators, prepositions, verbs, numeratives, and morphemes in addition to the original pronoun/determinative split. Reason: the old filename no longer described the scope and was likely to cause drift.
- **Superseding decision on `so/SCONJ`: auto-tag it as `preposition[clausal]`.** The current project treats UD `SCONJ so` as purpose/result `so`, which patterns with a preposition taking a clausal complement; this removes a known-resolved item from the review queue without affecting coordinator `so`, which is still outside the current automatic layer.
- **Superseding decision on UD coordinator-like `as well as`: auto-tag it as `coordinator[additive]`.** In EWT, the live pattern is tightly delimited: `as/ADV` with `deprel=cc`, `ExtPos=CCONJ`, and fixed `well as`. Those are ordinary coordinator uses rather than unresolved boundary cases, so they should not remain in review.
- **Superseding decision on UD coordinator-like `yet`: auto-tag it as `coordinator[adversative]`.** In EWT, the live pattern is uniformly `yet/CCONJ` with `deprel=cc`, including sentence-initial adversative uses. Those are ordinary coordinator uses, not a residual boundary class for the current rule layer.
- **Superseding decision on UD coordinator-like `rather than`: auto-tag it as `coordinator[rather_than]`.** In EWT, the `cc` pattern is cleanly delimited: `rather/ADV` with `deprel=cc`, `ExtPos=CCONJ`, and fixed `than`. Those alternation/preference uses are coordinators in the current layer.
- **Superseding decision on `along with`: auto-tag it as `preposition`.** In EWT, the live `along with` pattern is uniformly prepositional, not coordinator-like: most tokens are `along/ADP` with `deprel=case`, and the remaining two train tokens are `along/ADP` with `deprel=mark` introducing participial clauses. NP-complement uses are now treated as `preposition[complex_with]`, and clause-introducing uses as `preposition[clausal]`.
- **Superseding decision on `whatever`: auto-tag it as `determinative`.** The EWT residue covered headed and fused-head uses, but the user’s analysis is that these are all determinatives. The rule layer now normalizes `whatever` across UD `DET` and `PRON` tags rather than keeping it in boundary review.
- **Superseding decision on DET-tagged `quite`: auto-tag it as `adverb[degree]`.** The user’s decision is that predeterminer is a syntactic function, not a lexical category. EWT’s DET/PDT-tagged `quite` tokens are therefore retagged as adverbs rather than left unresolved.
- **Superseding decision on dialectal determiner `them`: auto-tag it as `determinative`.** In non-standard determiner use such as “them lovely kebabs,” the category is determinative, not pronoun. The sidecar no longer keeps this single EWT token in review.
- **Superseding decision on expressive `wtf`: auto-tag it as a headless determinative.** The user’s analysis treats `wtf` as the written string “what the fuck,” not as a separate spoken lexical word. The output layer therefore normalizes it to the same determinative space as headless `what`.
- **Superseding decision on `et` in `et al.`: auto-tag it as `coordinator[additive]`.** This remains peripheral and formulaic, but the categorial decision is clear enough that the single EWT token no longer needs review.
- **Superseding decision on clause-marking `rather than`: auto-tag it as `preposition[clausal]`.** The construction in examples like “rather than undermining Afghanistan’s stability ...” is not a coordinator. In the current CGEL/UD alignment layer it is treated like other clause-introducing markers outside the small dedicated subordinator class: a preposition with a clausal complement.
- **Low-frequency GUM lexical residue that instantiates existing categories is auto-tagged rather than left in fallback.** `s/he` is treated as a personal pronoun, `oneself` as reflexive, `whosoever` as a wh-pronoun, `yonder` as a demonstrative determinative, and foreign `une` as an article-like determinative in the narrow `DET + X` title pattern. Reason: these are not new category questions, only sparse lexical gaps that otherwise create avoidable review noise outside EWT.
- **Non-existential `there` is auto-tagged as `preposition[intransitive]`, including disfluent reparandum uses.** Reason: the user’s analysis is that `there` is pronominal only in existentials; once `there-expletive` handles the existential case, the remaining demonstrative/locative uses belong with the intransitive prepositions even when the utterance is interrupted.
- **Tokenized `self` in `self-` compounds is not a pronoun.** Forms like `self-deprecating` are treated as compounds with `self` as the first component rather than as syntactic pronoun uses, so the sidecar now maps tokenized `self` in this pattern to `noun[self_compound_initial]`. Reason: the contrast is morphological, not the ordinary reflexive-pronoun construction.
- **`Mat` in `Mat Fereder` is treated as out-of-scope foreign residue, not as English review burden.** The sidecar uses the narrow category `x[foreign_component]`, but it is no longer marked `needs_review=true`. Reason: the user explicitly ruled that this foreign token is not relevant to the English retagging task, so it should not remain in review counts once the known-wrong determinative analysis has been removed.
- **`littleprince` runs cleanly under the existing stabilized rule table and does not justify corpus-specific rule additions.** The first pass over `en_littleprince-ud-test` produced `2,032` auto-tagged rows and `0` review rows. Reason: this confirms the current EWT/GUM-derived inventory generalizes to another overt-form corpus, so the next effort should move to a corpus that is more likely to expose new gaps rather than polishing `littleprince`.
- **GUMReddit needs a structural fallback layer because its core FORM/LEMMA columns are delexicalized.** The corpus stores `_` in the main lexical columns, so lexeme-driven rules cannot fire reliably. The current rule table therefore adds low-priority structural rules for recoverable categories such as articles, demonstratives, quantificational determinatives, personal pronouns, possessives, reflexives, indefinites, and expletives. Reason: this preserves the EWT/GUM lexical analysis while still making GUMReddit partially usable instead of collapsing almost entirely into fallback review.
- **Do not overfit delexicalized GUMReddit beyond what the remaining signal supports.** After the structural fallback layer, the residual GUMReddit review burden is dominated by unresolved wh-items and a few miscellaneous delexicalized forms whose lexical identity is not recoverable from the available columns. Reason: without surface lexemes, further automatic normalization would increasingly encode guesses rather than stable analyses.
- **Delexicalized MISC hints only backfill missing lexical identity; they do not override overt FORM/LEMMA in ordinary corpora.** The retagger now consults `Lem`, `CorrectForm`, `MSeg`, `Len`, and `ref`-bearing enhanced dependencies when raw lexical columns are `_`, but concrete EWT/GUM forms and lemmas still win. Reason: this sharply reduces GUMReddit review burden without letting typo metadata destabilize the already-stable lexical corpora.
- **A small delexicalized wh-recovery layer is now justified for GUMReddit.** Segmented `what-ever`, short `WP` relatives with `ref`, and `WDT` relatives whose length cleanly identifies `that` vs `which` are now auto-tagged. Reason: these are not speculative guesses once the available metadata is combined, and they remove most of the remaining GUMReddit review load at low risk.
- **GUMReddit is now frozen as partial coverage rather than a finishable target under the current data.** The current structural plus hint-based layer extracts most of the recoverable signal, but the remaining residue is dominated by missing lexical identity rather than weak rules. Reason: with delexicalized surface columns, further auto-tagging would mostly be guesswork unless undelexicalized text or token alignment becomes available.
- **The next corpus extensions should prioritize overt lexical forms, starting with the smallest clean wins.** `en_pronouns-ud-test` and `en_pud-ud-test` were extended immediately after freezing GUMReddit and both now run cleanly with 0 review rows. Reason: once GUMReddit was recognized as structurally limited, the best use of effort was to resume expansion on corpora where the lexeme-driven rule table can work normally.
- **Add narrow lexical gap-fill rules only when they represent real category knowledge rather than corpus-specific guessing.** The current examples are bare `one/PRON` as generic pronoun when UD omits `PronType=Prs`, relative `that` recoverable from a `ref` dependency even when `PronType=Rel` is missing, and `where/PRON` as an intransitive preposition. Reason: these close genuine rule gaps exposed by overt-form corpora without destabilizing the already-covered corpora.
- **Correct the `data_raw/` symlink layer rather than relying only on direct `../../corpora/...` paths.** The original symlinks were off by one directory and did not resolve. Reason: the repo documentation and local workflow both assume `data_raw/` is a working mirror of the corpus inventory.
- **Dependent possessive `DET` tags do not change the target category.** When UD tags forms like `my`, `your`, `their`, `its`, `his`, or `her` as `DET`, the sidecar still maps them to `pronoun[possessive]`. Reason: the user’s analysis is that these are pronouns even in dependent determiner function, and ATIS/CTeTex exposed this as a real cross-corpus gap.
- **Non-existential `there` is now auto-tagged as `preposition[intransitive]` without a feature guard.** Expletive `there` is still protected by the higher-priority `there-expletive` rule. Reason: ATIS showed that some corpora misfeature locative `there`, but the categorial decision is still clear once existential uses are excluded.
- **Temporal `when` is treated like locative `where`: `preposition[intransitive]`.** Reason: the project’s current CGEL alignment treats these wh-items as prepositions rather than pronouns/adverbs, and CTeTex exposed a small but clean temporal-relative gap.
- **`whichever` is auto-tagged as a determinative.** Reason: this is parallel to the existing `whatever` decision and removed a known-resolved fused-head case from CTeTex review.
- **Relative `that` and `which` can be recovered structurally from relative-clause attachment, including through an intervening `xcomp`.** Reason: CTeTex exposed relative pronouns attached below the `acl:relcl` head, so the structural recovery rule now also consults the grandparent dependency rather than only the immediate head.
- **Leave `other/DET` as explicit review residue for now.** CTeTex is now down to three such tokens, but the project has not yet fixed a lexical-category decision for prenominal `other`. Reason: this is a genuine categorial decision point, not just a missing lexical rule.
- **Superseding decision on `other`: auto-tag it as `adjective`.** Prenominal and fused-head uses of `other` are treated as adjectives rather than determinatives. Reason: the user fixed the lexical category explicitly, and this removed the final three CTeTex review items.
- **Superseding decision on compound indefinite items like `someone`, `anything`, and `nothing`: auto-tag them as `determinative[indefinite]`.** These are not pronouns in the target inventory for this project. Reason: the user explicitly corrected the lexical inventory, and the change applies cleanly across EWT, GUM, GENTLE, PUD, CTeTex, and the structural fallback layer for GUMReddit.
- **Superseding decision on `half`: treat it as `noun[fractional]`, even when UD tags it `DET` or `ADJ`.** Reason: this aligns the sidecar with the current numerative analysis and removes a real cross-corpus mismatch rather than leaving `half` inside the determiner layer.
- **Superseding decision on `such`: auto-tag it as `adjective[such]`.** Reason: the user explicitly rejected a determinative analysis here; predeterminer behavior is syntactic function, not lexical category.
- **Superseding decision on `yonder`: default to `preposition[intransitive]`, retaining only marginal `DET` uses as `determinative[demonstrative]`.** Reason: the user’s analysis is that `yonder` is normally parallel to locative `there`, with a real but marginal determiner use in examples like “yonder hills.”
